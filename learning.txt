 ["What I have learned from this course"] 
 
- Navigating files in Git Bash – Using editors like vim to modify files.
- Branch management – Creating and switching between branches.
- Pull requests – Merging changes efficiently using GitHub UI and CLI.
- Handling errors – Fixing issues like read-only files (E45 error) and repository access problems.
- Repository forking – Copying projects for independent modifications.
- Undoing actions – Reverting a git clone operation.
- Renaming branches – Changing master to main for better project organization.
